# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
# Install dependencies
pip install -r requirements.txt

# Run full benchmark (all 3 architectures)
python scripts/run_benchmark.py

# Run a single architecture
python scripts/run_benchmark.py --arch rag
python scripts/run_benchmark.py --arch agentic
python scripts/run_benchmark.py --arch hybrid

# Skip LLM-as-judge scoring (faster iteration)
python scripts/run_benchmark.py --no-judge

# Generate Markdown report from saved results
python scripts/generate_report.py
python scripts/generate_report.py --run results/run_20240427_120000.json
```

Required environment variable: `ANTHROPIC_API_KEY`. Optional: `CORPUS_DIR`, `INDEX_DIR` (default to `../RAG_finances/data/corpus` and `../RAG_finances/data/index`).

## Architecture

This project benchmarks three retrieval strategies for financial Q&A against 10 queries in `benchmark/eval_set.json`. All LLM calls use Claude Haiku 4.5 with prompt caching on the system prompt.

### Three Architectures

All three inherit `BaseArchitecture` (`architectures/base.py`) and return a `BenchmarkRun` dataclass.

- **Pure RAG** (`architectures/pure_rag.py`): Hybrid search → RRF fusion → cross-encoder rerank → single LLM call with formatted context.
- **Pure Agentic** (`architectures/pure_agentic.py`): LLM agent loop (max 8 iterations) with `grep_corpus` and `read_article` tools; no pre-retrieval.
- **Hybrid** (`architectures/hybrid.py`): RAG pre-retrieval provides initial context; agent can optionally call tools to supplement.

### Shared Components

- **`shared/llm.py`** — Anthropic client wrapper; handles prompt caching, token counting, and cost calculation. All LLM calls go through here.
- **`shared/search.py`** — `HybridSearcher` (Qdrant dense + BM25 sparse + RRF fusion) and `Reranker` (cross-encoder). Returns `SearchResult` dataclass.
- **`shared/corpus_tools.py`** — `CorpusIndex` holds in-memory corpus; `run_tool()` dispatches `grep_corpus`/`read_article` calls from Claude. `TOOLS` defines the JSON schemas for Claude's tool use.
- **`shared/chunker.py`** — Structural markdown chunker (heading split + size split with overlap); used when building the index.

### Configuration

All tunable parameters live in `config.py`: embedding/reranker model names, retrieval top-K values, RRF_K, context budget (`MAX_CONTEXT_TOKENS = 6000`), agent iteration cap (`AGENTIC_MAX_ITERATIONS = 8`), and Haiku 4.5 pricing for cost estimation.

### Evaluation & Output

- **`benchmark/runner.py`** — Orchestrates runs, computes heuristic metrics (topic coverage, ticker recall, citation count), saves JSON to `results/`.
- **`benchmark/judge.py`** — LLM-as-judge scoring 4 criteria (completeness, groundedness, citation quality, conciseness) on a 1–5 scale.
- `REPORT.md` is auto-generated by `scripts/generate_report.py` from the saved JSON results.

# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
# Install dependencies
pip install -r requirements.txt

# Ask a single question (defaults to rag; use --architectures to pick)
python scripts/run_benchmark.py --query "What was Abbott's Q4 dividend?"
python scripts/run_benchmark.py --query "..." --architectures agentic
python scripts/run_benchmark.py --query "..." --architectures rag hybrid  # side-by-side

# Run full benchmark (all 3 architectures)
python scripts/run_benchmark.py

# Run specific architectures
python scripts/run_benchmark.py --architectures rag hybrid
python scripts/run_benchmark.py --architectures agentic

# Skip LLM-as-judge scoring (faster iteration)
python scripts/run_benchmark.py --no-judge

# Skip smoke-test confirmation prompt
python scripts/run_benchmark.py --no-smoke

# Generate Markdown report from saved results (auto-detects results/runs/ or results/smoke/)
python scripts/generate_report.py

# Point at a specific results directory
python scripts/generate_report.py --baseline results/smoke

# Generate eval set (produces benchmark/eval_set_v2.json)
python scripts/generate_eval.py

# Build corpus index (INDEX.md summaries)
python scripts/build_index.py
```

Required environment variable: `ANTHROPIC_API_KEY`. Optional: `CORPUS_DIR`, `INDEX_DIR` (default to `../RAG_finances/data/corpus` and `../RAG_finances/data/index`).

## Architecture

This project benchmarks three retrieval strategies for financial Q&A against queries in `benchmark/eval_set_v2.json` (20 queries across 4 categories; falls back to `eval_set.json`). All LLM calls use Claude Haiku 4.5 with prompt caching on the system prompt.

### Three Architectures

All three inherit `BaseArchitecture` (`architectures/base.py`) and return a `BenchmarkRun` dataclass.

- **Pure RAG** (`architectures/pure_rag.py`): Hybrid search → RRF fusion → cross-encoder rerank → single LLM call with formatted context.
- **Pure Agentic** (`architectures/pure_agentic.py`): LLM agent loop (max 8 iterations, max 10 tool calls) with `grep_corpus` and `read_article` tools; no pre-retrieval.
- **Hybrid** (`architectures/hybrid.py`): RAG pre-retrieval provides initial context; agent can optionally call tools to supplement.

### Shared Components

- **`shared/llm.py`** — Anthropic client wrapper; handles prompt caching, token counting, and cost calculation. All LLM calls go through here.
- **`shared/search.py`** — `HybridSearcher` (Qdrant dense + BM25 sparse + RRF fusion) and `Reranker` (cross-encoder). Returns `SearchResult` dataclass.
- **`shared/corpus_tools.py`** — `CorpusIndex` holds in-memory corpus; `run_tool()` dispatches `grep_corpus`/`read_article` calls from Claude. `TOOLS` defines the JSON schemas for Claude's tool use.
- **`shared/chunker.py`** — Structural markdown chunker (heading split + size split with overlap); used when building the index.

### Configuration

All tunable parameters live in `config.py`: embedding/reranker model names, retrieval top-K values, RRF_K, context budget (`MAX_CONTEXT_TOKENS = 6000`), agent iteration cap (`AGENTIC_MAX_ITERATIONS = 8`), tool call cap (`AGENTIC_MAX_TOOL_CALLS = 10`), and Haiku 4.5 pricing for cost estimation.

### Evaluation & Output

- **`benchmark/runner.py`** — Orchestrates runs, calls judge, saves JSONL to `results/runs/<arch>_<timestamp>.jsonl`.
- **`benchmark/judge.py`** — LLM-as-judge scoring 4 fields via tool use: `faithfulness` (0–1), `correctness` (0–1), `hallucination_count` (int), `refusal_correct` (bool). Results cached to `results/.judge_cache.json`.
- **`benchmark/metrics.py`** — `compute_aggregate` and `compute_by_category` over result rows.
- `REPORT.md` is auto-generated by `scripts/generate_report.py`. It auto-detects `results/runs/` first, then falls back to `results/smoke/`.
